 CFPB Consumer Complaints Dashboard – Trend and Analysis (Excel)

This project presents a descriptive analysis of consumer complaint data based on the Consumer Financial Protection Bureau (CFPB) dataset from the DataDNA Onyx Data Challenge. The dashboard was developed in Microsoft Excel to organize and visualize complaint records, focusing on summarizing historical data and displaying patterns in complaint volume, company involvement, response behavior, and resolution timelines. The dataset contains information on companies, products, sub-products, complaint submission methods, response types, response timeliness, and geographic distribution across U.S. states, with a total of 62,516 complaints and 1,081 companies represented in the analysis.

The dashboard displays the distribution of complaints across companies, highlighting variation in complaint counts among different organizations. It presents complaint volumes by product and sub-product categories, showing how complaints are distributed across financial services such as credit cards, mortgages, loans, and money transfers. The visualization also includes complaint issues, allowing the observation of how frequently specific problem types occur within the dataset. In addition, complaint submission channels such as web, referral, phone, and email are represented to show how complaints are received.

Time-based patterns are included through trend visualizations that display complaint submission counts and response activity over multiple years, illustrating how volumes change over time. The dashboard also summarizes response behavior by showing how companies respond to consumers, including categories such as closed with explanation, closed with monetary relief, and other response types. Timeliness of responses is presented to show the distribution of whether responses were provided on time or delayed, along with calculated average response time.

Geographic distribution is incorporated using state-level complaint counts, enabling observation of how complaints are spread across different regions. Regional aggregation further groups complaints into broader geographic areas, allowing comparison of complaint volumes between regions. Interactive filtering through slicers allows users to explore the data dynamically by state and product, updating all visualizations accordingly.

This project follows a descriptive analytics approach, focusing only on summarizing and presenting existing data without applying predictive models or interpretations. The purpose of the dashboard is to transform raw complaint records into structured visual insights, enabling clear observation of distributions, comparisons, and trends across multiple dimensions. The entire analysis was conducted using Microsoft Excel, utilizing pivot tables, charts, and slicers to build an interactive reporting interface suitable for data exploration and presentation. This project is intended for educational and portfolio purposes as part of the DataDNA Onyx Data Challenge.
